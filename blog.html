Tutorial de Git: Desde lo Básico hasta Colaboración Remota

Fecha: Octubre 2024

En este tutorial, aprenderemos los fundamentos de Git, una herramienta esencial para el control de versiones en el desarrollo de software. Cubriremos desde la inicialización de un repositorio local hasta la conexión con un repositorio remoto en GitHub.

1. Inicialización del Repositorio Localmente

Para comenzar a usar Git en tu proyecto, primero necesitas inicializar un repositorio local. Sigue estos pasos:

  1. Abre tu terminal y navega al directorio de tu proyecto.
  2. Ejecuta el siguiente comando:
git init

Este comando crea un nuevo subdirectorio .git que contiene todos los archivos necesarios para el repositorio.

Inicialización del repositorio Git

2. Creación de Branches

Las ramas (branches) en Git te permiten desarrollar funcionalidades aisladas unas de otras. Aquí te mostramos cómo crearlas y utilizarlas:

  1. Para crear una nueva rama, usa:
git branch nombre-de-la-rama
  1. Para cambiar a la nueva rama:
git checkout nombre-de-la-rama
  1. O puedes crear y cambiar a la nueva rama en un solo comando:
git checkout -b nombre-de-la-rama

Ejemplo Práctico con Merge

Supongamos que estás trabajando en una nueva función llamada "login":

  1. Crea y cambia a una nueva rama:
git checkout -b feature-login
  1. Realiza tus cambios y haz commits en esta rama.
  2. Cuando la función esté lista, vuelve a la rama principal:
git checkout main
  1. Fusiona (merge) la rama de la función con la principal:
git merge feature-login

Proceso de merge entre branches

3. Agregar un Repositorio Remoto

Para colaborar con otros desarrolladores, necesitas conectar tu repositorio local a uno remoto en GitHub. Sigue estos pasos:

  1. Crea un nuevo repositorio en GitHub.
  2. En tu repositorio local, agrega el repositorio remoto:
git remote add origin https://github.com/tu-usuario/tu-repositorio.git
  1. Verifica que se haya agregado correctamente:
git remote -v
  1. Sube tus cambios al repositorio remoto:
git push -u origin main
